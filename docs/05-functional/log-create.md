# 活動ログ作成

ユーザーは、自分の活動をログとして作成できる。

## 優先度

Must

## 要件

* 活動のタイトルを登録できる
* 活動日を登録できる
* MVPでは活動日は日付のみを扱う
* 未来日の活動日も登録できる
* 活動種別を任意で登録できる
* 活動の対象名を任意で登録できる
* タグを任意で複数登録できる
* 外部URLを任意で複数登録できる
* 短いメモを任意で登録できる
* ネタバレを含むかどうかを設定できる
* 活動ログをAT Protocol上のCumulog独自レコードとして保存できる
* 活動ログが公開データとして保存されることを確認できる
* 公開してよい活動ログとして保存できる
* 保存に失敗した場合、保存が成功したように表示しない
* 保存に失敗した場合、入力済み内容を可能な限り保持し、ユーザーが再試行できるようにする

## 入力項目

| 項目     | 必須 / 任意 | MVPでの扱い                       |
| ------ | ------- | ----------------------------- |
| タイトル   | 必須      | 活動ログの見出しとして登録する               |
| 活動日    | 必須      | MVPでは日付のみを扱う。未来日も登録できる        |
| 活動種別   | 任意      | 自由入力として扱う                     |
| 対象名    | 任意      | MVPでは1件のみ扱う                   |
| タグ     | 任意      | 自由入力、複数登録可                    |
| 外部URL  | 任意      | 複数登録可。http / https のURLを対象とする |
| メモ     | 任意      | 短文メモとして扱う                     |
| ネタバレ有無 | 必須      | 未選択時は「ネタバレなし」として扱う            |

## 入力制約の方針

* タイトル、メモ、タグ、対象名、活動種別、外部URLには、Cumulogの用途に合った入力制約を設ける
* メモは長文記事の代替ではなく、活動ログを補足する短文として扱う
* 空のタグは登録できない
* 1つの活動ログ内で同一のタグを重複登録しない
* 外部URLは、MVPでは http または https のURLを登録対象とする
* 具体的な文字数上限、URL件数上限、タグ件数上限、バリデーション条件は基本設計以降で定義する

## 受け入れ基準

* タイトルと活動日を入力して保存すると、AT Protocol上にCumulog活動ログレコードが作成され、Cumulogの一覧に表示される
* 保存前に、保存内容と公開データとして扱われる旨を確認できる
* 未来日の活動日を入力して保存できる
* 保存に失敗した場合、保存が成功したように表示されない
* 保存に失敗した場合、入力済み内容が可能な限り保持され、ユーザーが再試行できる
