# アカウント連携

ユーザーは、Bluesky公式PDSを利用するAT ProtocolアカウントでCumulogを利用できる。

## 優先度

Must

## 要件

* Bluesky公式PDSを利用するAT Protocolアカウントで利用を開始できる
* MVPでは、任意PDSへの対応は対象外とし、将来検討とする
* ログイン状態を確認できる
* ログアウトできる
* 自分のAT Protocolアカウントに紐づく活動ログを扱える
* MVPでは、自分の活動ログのみを閲覧・操作対象とする
* 認証失効または権限不足により操作できない場合は、ユーザーに理由を分かる形で表示する
* 対応外のアカウントまたはPDSで利用できない場合は、その理由をユーザーに表示する
* Cumulogは、活動ログの作成・閲覧・編集・削除に必要な範囲の権限のみを要求する
* ユーザーに不要な権限を要求しない

## ユーザー識別方針

* ユーザー表示にはhandleを用いてよい
* 活動ログ所有者の識別はDIDを前提とする
* handleが変更された場合でも、自分の活動ログの所有関係が破綻しないようにする

## 基本設計以降で定義すること

* 認証方式の具体的な実装
* 要求する権限スコープ
* セッション管理
* 認証失効時の詳細な挙動
* 対応PDS判定の具体的な方法
* 連携解除・利用終了時の具体的な手段（方針は[公開・データ方針](../03-publication-policy.md)を参照）

## 受け入れ基準

* Bluesky公式PDSを利用するAT ProtocolアカウントでCumulogを利用開始できる
* 対応外のアカウントまたはPDSの場合、利用できない理由を確認できる
* ログアウトできる
* 認証失効または権限不足により操作できない場合、ユーザーに理由が表示される
