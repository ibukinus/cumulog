# 入力項目の考え方

活動ログは、少ない必須項目で作成できる。

## 優先度

Must

## 要件

* 活動ログは短時間で作成できる
* 必須項目を増やしすぎない
* 活動日とタイトルを基本的な必須項目とする
* ネタバレ有無はMVPで扱い、未選択時は「ネタバレなし」として扱う
* 活動種別、対象名、タグ、外部URL、メモは任意で登録できる
* 感情タグは初期リリースでは後回し可能とする
* 長文感想はCumulog内に書くのではなく、外部URLに紐づける前提とする
* 具体的な文字数上限、URL件数上限、タグ件数上限、バリデーション条件は基本設計以降で定義する
