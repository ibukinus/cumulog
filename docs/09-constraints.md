# 制約

## 開発上の制約

* 個人開発で実装・運用できる範囲に収める
* MVPでは機能を増やしすぎない
* 運用負荷の高い機能は後回しにする
* モデレーション負荷の高い機能は後回しにする
* AT Protocol上のCumulog独自レコード保存をMVPの中核とする
* 活動ログの正本はAT Protocol上のCumulog独自レコードとする
* MVPではBluesky公式PDSを利用するAT Protocolアカウントを主対象とする
* 任意PDS対応は将来検討とする
* Cumulog独自のLexicon設計は基本設計以降で具体化する
* Cumulogは所有ドメインmp0.jpのサブドメイン（cumulog.mp0.jp）で運用する
* LexiconのNSIDはmp0.jp配下（jp.mp0.cumulog.*）を前提とする

## プロダクト上の制約

* 長文投稿サービスを目指さない
* 汎用SNSを目指さない
* 予定管理サービスを目指さない
* すべてのジャンルに最適化した詳細機能は持たない
* 公開ログ中心のサービスとして扱う
* 非公開メモや下書きはMVPの対象にしない
* Bluesky投稿は活動ログ保存とは別の共有機能として扱う
* MVPではCumulog画面上で他ユーザーの公開ログを閲覧する機能は持たない
* MVPでは通知、リマインダー、カレンダー同期、予定の自動更新は対象外とする

## 利用上の制約

* ユーザーが公開してよい内容を記録することを前提とする
* 未来日の活動ログも登録できるが、Cumulogは予定の実行や通知を保証しない
* ネタバレを含む活動ログには明示できることを前提とする
* 外部URLの内容自体はCumulog側で管理しない
* 外部URL先の内容、安全性、公開状態はCumulogでは保証しない
* 外部サービスの仕様変更により、一部の連携体験が変わる可能性がある
* 一度公開された情報は、外部サービスや他クライアントに取得済みである可能性がある
* Cumulog画面上で他ユーザーのログを表示しない場合でも、AT Protocol上では公開データとして参照される可能性がある
