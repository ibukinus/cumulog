# 成功条件

MVPの成功条件は以下とする。

## 利用価値の確認

* ユーザーが活動ログを継続して登録できる
* 登録した活動ログをあとから見返せる
* 外部URL付きの活動ログを残せる
* タグによって過去の活動を探せる
* ネタバレ有無を明示して活動ログを保存できる
* 活動ログがAT Protocol上の公開データとして保存されることに価値を感じられる
* 未来日の活動ログも記録できる
* 未来日の活動ログを扱っても、Cumulogが予定管理サービスでないことがユーザーに伝わる

## 個人利用での成立

* 開発者自身が2週間以上継続して利用できる
* 30件以上の活動ログを登録しても一覧・年表で振り返りやすい
* 30件以上の活動ログを登録した状態で、作成・閲覧・編集・削除・タグ絞り込み・外部URL参照が破綻なく利用できる
* noteやブログなど外部サービスへの索引として機能する
* AT Protocol上に活動ログを残すことに価値を感じられる
* Bluesky共有がなくても、活動ログ単体で価値を感じられる

## MVPとしての成立

* Bluesky公式PDSを利用するAT Protocolアカウントで利用できる
* ログイン状態を確認できる
* ログアウトできる
* 活動ログを作成、閲覧、編集、削除できる
* 活動ログをAT Protocol上のCumulog独自レコードとして保存できる
* 活動ログの正本をAT Protocol上のCumulog独自レコードとして扱える
* 自分の活動年表として利用できる
* 活動日の日付降順で活動ログを確認できる
* 同一活動日の活動ログは作成日時降順で確認できる
* 未来日の活動ログを登録できる
* タグで過去の活動ログを絞り込める
* MVPでは単一タグの完全一致で絞り込める
* 複数の外部URLを活動ログに紐づけられる
* 外部URLは http / https を対象にできる
* 登録できないURL形式の場合、保存前にエラーとして確認できる
* ネタバレ有無を設定できる
* ネタバレ有無をラベルとして確認できる
* 公開してよい活動ログを扱える
* 公開前提であることをユーザーに明示できる
* 削除しても外部に残る可能性があることをユーザーに明示できる
* 認証失効、権限不足、保存失敗、編集失敗、削除失敗、一覧取得失敗、詳細取得失敗をユーザーに分かる形で表示できる
* 利用開始前に利用規約とプライバシーポリシーを確認できる
* 個人開発として継続運用できる
