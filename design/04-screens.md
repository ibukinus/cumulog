# 画面設計

スマートフォンブラウザでの利用を第一に、単一カラムを基本とする。

## 画面一覧

| # | 画面 | 主な役割 |
| --- | --- | --- |
| S1 | ランディング / オンボーディング | サービス説明・公開前提の明示・規約確認・ログイン導線 |
| S2 | ログイン | handle入力とOAuth認可への遷移、対応外PDSの理由表示 |
| S3 | 活動ログ一覧（年表） | 日付降順の一覧、タグ・感情タグ・活動種別・対象名・年月による絞り込み、新規作成導線 |
| S4 | 活動ログ詳細 | 全項目の表示、編集・削除導線 |
| S5 | 活動ログ作成 | 入力フォームと保存前の公開前確認 |
| S6 | 活動ログ編集 | 入力フォームと公開前提の再明示 |
| D1 | 削除確認ダイアログ | 削除前の注意表示 |
| D2 | Bluesky共有ダイアログ | 投稿文面の確認・編集と投稿実行 |
| S7 | 設定 | ログイン状態表示、ログアウト、規約・ポリシーへのリンク、連携解除の案内 |
| S8 | 公開共有ページ | 共有URLからの単一活動ログの未認証閲覧 |

## 画面遷移

```
S1 ランディング ──規約確認→ S2 ログイン ──OAuth認可→ S3 一覧（年表）
                                              │
        ┌─────────────┬─────────────┼──────────────┐
        ↓             ↓             ↓              ↓
   S5 作成 ─保存→ S3   S4 詳細 ─→ S6 編集 ─保存→ S4   S7 設定 ─ログアウト→ S1
                        │
                        └─→ D1 削除確認 ─削除→ S3
```

* 未認証で S3 以降へアクセスした場合は S1 へ誘導する（S8 公開共有ページを除く）
* 初回ログイン直後は S3 に遷移し、活動ログ0件の場合は最初の作成へ誘導する導線（空状態からの作成ボタン）を表示する（[初回利用](../docs/05-functional/onboarding.md)）

## S1 ランディング / オンボーディング

* Cumulogが公開活動ログサービスであること、活動ログがAT Protocol上の公開データとして保存されることを、ログイン前に本文で明示する
* MVPの対応対象がBluesky公式PDSのアカウントであることを明示する
* 利用規約とプライバシーポリシーへのリンクを表示し、「利用開始により規約に同意したものとする」形式ではなく、ログイン操作の直前に確認できる配置とする
* 複雑なチュートリアルは置かない

利用規約・プライバシーポリシーの文面は、静的ページとして同サイト内に置く。文面の作成は実装時の作業とし、少なくとも「公開データとして保存されること」「削除しても外部に残る可能性があること」「Cumulogが保持するデータの範囲」を含める。

## S3 活動ログ一覧（年表）

* 活動日の日付降順、同一活動日内は作成日時降順で表示する
* 各行に、活動日・タイトル・活動種別・対象名・タグ・感情タグ・外部URLの有無（アイコン）・ネタバレラベルを表示する。感情タグは通常のタグと区別できる表示とする
* 未来日の活動ログも同じ一覧に表示し、未来日であることが分かる表示（「予定」ではなく未来日の日付表示のまま）とする
* タグをタップすると、そのタグで絞り込んだ一覧に切り替わる。絞り込み中は選択中のタグと解除操作を表示する。感情タグも同様に、タップでその感情タグによる絞り込みに切り替わる（値の完全一致によるクライアント側フィルタ）
* 活動種別・対象名も同様に、行内の表示をタップするとその値で絞り込んだ一覧に切り替わる。絞り込みは値の完全一致によるクライアント側フィルタとする（[データフロー設計](05-data-flow.md)）
* 活動日をタップすると、その活動日が属する年月で絞り込んだ一覧に切り替わる。年月絞り込みは活動日（`YYYY-MM-DD`）の年月部分（`YYYY-MM`）の一致によるクライアント側フィルタとする
* 絞り込み条件は同時に1つ（タグ・感情タグ・活動種別・対象名・年月のいずれか）とし、別の条件を選ぶと置き換える。絞り込み中は条件の種類と値、解除操作を表示する
* 絞り込み結果が0件の場合は、絞り込みの解除を促す空状態を表示する
* 取得失敗時は、0件表示と区別したエラー表示と再試行操作を表示する（[エラー処理設計](06-error-handling.md)）

## S4 活動ログ詳細

* 1件の記録として読める構成で表示する。項目名と値の列挙ではなく、活動日・活動種別・タイトル・対象名を見出し部に、メモを本文、タグ・外部URLをそれに続く要素として配置する
* 登録済みの全項目（活動日・タイトル・活動種別・対象名・タグ・感情タグ・メモ・外部URL・ネタバレ）を確認できる。値が未設定の任意項目は項目名を含めて表示しない
* 感情タグは通常のタグと区別できる表示とする（[要件: 感情タグ](../docs/05-functional/emotion-tag.md)）
* ネタバレはラベルとして見出し付近に表示する。「なし」はラベルを表示しない（[デザイン言語](09-design-language.md)の状態表現に従う）
* 「AT Protocol上の公開レコードとして保存されている」ことを示す表示を置く（レコードのat-uri表示を含む）
* 外部URLはリンクとして表示し、タップで新しいタブで開く（[セキュリティ設計](07-security.md)）
* 「Blueskyで共有」操作を置き、D2 Bluesky共有ダイアログを開く。共有導線はMVPでは詳細画面のみとする（要件「作成時または作成後に選べる」は作成後＝詳細画面からの共有で満たす）
* 「読み込めない活動ログ」には共有操作を表示しない

## S5 作成 / S6 編集

* 入力項目と必須・任意は[活動ログ作成](../docs/05-functional/log-create.md)の入力項目表に従う
* 活動種別は自由入力とし、入力補助として自分の既存ログで使用した活動種別を候補表示する（候補は取得済み一覧から生成し、固定マスタは持たない）
* タグも同様に、自分の既存ログで使用したタグを入力候補として表示する
* 感情タグはタグとは別の入力欄とし、自分の既存ログで使用した感情タグを入力候補として表示する（固定の感情マスタは持たない）
* ネタバレは「ネタバレなし」「軽微なネタバレ」「重大なネタバレ」の3択とし、各選択肢に説明文を添える
  * ネタバレなし：作品や展開の核心に触れない
  * 軽微なネタバレ：見た人が推測できる程度の内容に触れる
  * 重大なネタバレ：結末や核心的な展開に触れる
* 説明文の文言は実装時に調整してよいが、3区分の意味は変えない

### 公開前確認

* 保存ボタンを押すと、即時保存せず確認ステップを表示する
* 確認ステップでは、保存内容の全項目と「この内容はAT Protocol上の公開データとして保存される」旨を表示する
* ネタバレを「軽微」「重大」に設定している場合は、ネタバレ設定も確認内容に含めて表示する
* 編集時は「編集後の内容も公開データとして扱われる」旨を表示する
* 確認ステップから入力へ戻って修正できる

## D1 削除確認ダイアログ

* 削除対象のタイトルと活動日を表示する
* 「読み込めない活動ログ」（[Lexicon・データ設計](02-lexicon.md)）はタイトル・活動日を持たない可能性があるため、代わりにレコードのrkeyを含むat-uriを削除対象の識別子として表示する
* 「削除しても、外部サービスや他のクライアントに取得済みの情報が残る可能性がある」旨を表示する
* 確定操作は取り消し操作と明確に区別する

## D2 Bluesky共有ダイアログ

* 活動ログの内容から生成した既定文面を編集可能な状態で表示する（[データフロー設計](05-data-flow.md)）
* Blueskyへの公開投稿であることを表示する
* 書記素数のカウントを表示し、上限（300書記素）を超える間は投稿できない
* 投稿の確定操作と取り消し操作を明確に区別する
* 投稿の成否は活動ログ保存の成否と区別して表示する。失敗時は編集した文面を保持したままダイアログ内に理由を表示する（[エラー処理設計](06-error-handling.md)）

## S8 公開共有ページ

* ルートは `/share/{did}/{rkey}` とし、認証の有無にかかわらず表示できる（[公開共有ページ](../docs/05-functional/share-page.md)）
* 活動ログの登録済みの全項目（活動日・タイトル・活動種別・対象名・タグ・感情タグ・メモ・外部URL・ネタバレ）と、所有者のhandle（解決できない場合はDID）を表示する。表示構成はS4と同じ「1件の記録として読める構成」とし、値が未設定の任意項目は表示しない
* 「AT Protocol上の公開データである」旨の表示を置く
* 閲覧専用とし、編集・削除・作成の操作は置かない。ログの操作導線は表示しない
* Cumulogの説明とランディング（S1）への導線を置く
* 存在しない・削除済みのログは「見つからない」表示、スキーマ検証を満たさないログと取得失敗は「表示できない」旨の表示とし、いずれも再試行または一覧等への誘導を含む（[エラー処理設計](06-error-handling.md)）
* 他ユーザーのログの探索・一覧・検索の導線は置かない

## 表示の共通方針

* 公開確認・削除確認・ネタバレラベル・入力エラーは、色だけに依存せず、文言とアイコンで内容が分かる表示とする（[非機能要件: アクセシビリティ](../docs/06-non-functional.md)）
* ネタバレラベルは一覧・詳細ともにテキストラベル（例：「重大なネタバレ」）で表示する。本文の非表示・折りたたみは初期リリースでは行わない
