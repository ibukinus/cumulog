# 画面設計

スマートフォンブラウザでの利用を第一に、単一カラムを基本とする。

## 画面一覧

| # | 画面 | 主な役割 |
| --- | --- | --- |
| S1 | ランディング / オンボーディング | サービス説明・公開前提の明示・規約確認・ログイン導線 |
| S2 | ログイン | handle入力とOAuth認可への遷移、対応外PDSの理由表示 |
| S3 | 活動ログ一覧（年表） | 日付降順の一覧、タグ絞り込み、新規作成導線 |
| S4 | 活動ログ詳細 | 全項目の表示、編集・削除導線 |
| S5 | 活動ログ作成 | 入力フォームと保存前の公開前確認 |
| S6 | 活動ログ編集 | 入力フォームと公開前提の再明示 |
| D1 | 削除確認ダイアログ | 削除前の注意表示 |
| S7 | 設定 | ログイン状態表示、ログアウト、規約・ポリシーへのリンク、連携解除の案内 |

## 画面遷移

```
S1 ランディング ──規約確認→ S2 ログイン ──OAuth認可→ S3 一覧（年表）
                                              │
        ┌─────────────┬─────────────┼──────────────┐
        ↓             ↓             ↓              ↓
   S5 作成 ─保存→ S3   S4 詳細 ─→ S6 編集 ─保存→ S4   S7 設定 ─ログアウト→ S1
                        │
                        └─→ D1 削除確認 ─削除→ S3
```

* 未認証で S3 以降へアクセスした場合は S1 へ誘導する
* 初回ログイン直後は S3 に遷移し、活動ログ0件の場合は最初の作成へ誘導する導線（空状態からの作成ボタン）を表示する（[初回利用](../docs/05-functional/onboarding.md)）

## S1 ランディング / オンボーディング

* Cumulogが公開活動ログサービスであること、活動ログがAT Protocol上の公開データとして保存されることを、ログイン前に本文で明示する
* MVPの対応対象がBluesky公式PDSのアカウントであることを明示する
* 利用規約とプライバシーポリシーへのリンクを表示し、「利用開始により規約に同意したものとする」形式ではなく、ログイン操作の直前に確認できる配置とする
* 複雑なチュートリアルは置かない

利用規約・プライバシーポリシーの文面は、静的ページとして同サイト内に置く。文面の作成は実装時の作業とし、少なくとも「公開データとして保存されること」「削除しても外部に残る可能性があること」「Cumulogが保持するデータの範囲」を含める。

## S3 活動ログ一覧（年表）

* 活動日の日付降順、同一活動日内は作成日時降順で表示する
* 各行に、活動日・タイトル・活動種別・対象名・タグ・外部URLの有無（アイコン）・ネタバレラベルを表示する
* 未来日の活動ログも同じ一覧に表示し、未来日であることが分かる表示（「予定」ではなく未来日の日付表示のまま）とする
* タグをタップすると、そのタグで絞り込んだ一覧に切り替わる。絞り込み中は選択中のタグと解除操作を表示する
* 取得失敗時は、0件表示と区別したエラー表示と再試行操作を表示する（[エラー処理設計](06-error-handling.md)）

## S4 活動ログ詳細

* 全項目（活動日・タイトル・活動種別・対象名・タグ・メモ・外部URL・ネタバレ）を表示する
* 「AT Protocol上の公開レコードとして保存されている」ことを示す表示を置く（レコードのat-uri表示を含む）
* 外部URLはリンクとして表示し、タップで新しいタブで開く（[セキュリティ設計](07-security.md)）

## S5 作成 / S6 編集

* 入力項目と必須・任意は[活動ログ作成](../docs/05-functional/log-create.md)の入力項目表に従う
* 活動種別は自由入力とし、入力補助として自分の既存ログで使用した活動種別を候補表示する（候補は取得済み一覧から生成し、固定マスタは持たない）
* タグも同様に、自分の既存ログで使用したタグを入力候補として表示する
* ネタバレは「ネタバレなし」「軽微なネタバレ」「重大なネタバレ」の3択とし、各選択肢に説明文を添える
  * ネタバレなし：作品や展開の核心に触れない
  * 軽微なネタバレ：見た人が推測できる程度の内容に触れる
  * 重大なネタバレ：結末や核心的な展開に触れる
* 説明文の文言は実装時に調整してよいが、3区分の意味は変えない

### 公開前確認

* 保存ボタンを押すと、即時保存せず確認ステップを表示する
* 確認ステップでは、保存内容の全項目と「この内容はAT Protocol上の公開データとして保存される」旨を表示する
* ネタバレを「軽微」「重大」に設定している場合は、ネタバレ設定も確認内容に含めて表示する
* 編集時は「編集後の内容も公開データとして扱われる」旨を表示する
* 確認ステップから入力へ戻って修正できる

## D1 削除確認ダイアログ

* 削除対象のタイトルと活動日を表示する
* 「削除しても、外部サービスや他のクライアントに取得済みの情報が残る可能性がある」旨を表示する
* 確定操作は取り消し操作と明確に区別する

## 表示の共通方針

* 公開確認・削除確認・ネタバレラベル・入力エラーは、色だけに依存せず、文言とアイコンで内容が分かる表示とする（[非機能要件: アクセシビリティ](../docs/06-non-functional.md)）
* ネタバレラベルは一覧・詳細ともにテキストラベル（例：「重大なネタバレ」）で表示する。本文の非表示・折りたたみは初期リリースでは行わない
