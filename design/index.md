# Cumulog 基本設計書

[要件定義書](../docs/index.md)にもとづき、CumulogのMVPの基本設計を定義する。

設計の対象は、要件定義の[今後の検討事項](../docs/10-open-issues.md)に挙げられた「基本設計以降で定義する」事項のうち、MVPの実装に必要な範囲とする。

初期リリース対象外の機能（Bluesky共有、感情タグ、月次振り返りなど）は、拡張余地の確認にとどめ、詳細設計は実装判断時に行う。

## 読み順

1. [全体構成](01-architecture.md) — システム構成・技術スタック・ホスティング
2. [Lexicon・データ設計](02-lexicon.md) — 活動ログレコードの項目・型・バリデーション
3. [認証・連携設計](03-auth.md) — OAuth、権限スコープ、対応PDS判定、セッション管理
4. [画面設計](04-screens.md) — 画面構成・画面遷移・主要UI方針
5. [データフロー設計](05-data-flow.md) — CRUD操作とXRPCの対応、一覧・絞り込み、整合性
6. [エラー処理設計](06-error-handling.md) — エラー分類・表示・再試行
7. [セキュリティ設計](07-security.md) — 不正入力対策・トークン管理
8. [テスト方針](08-test-policy.md) — 検証の進め方

## 要件定義の検討事項との対応

[docs/10-open-issues.md](../docs/10-open-issues.md) の各論点は、以下で扱う。

| 検討事項 | 対応箇所 |

| ActivityLog項目の型・文字数上限・バリデーション・Lexicon上の表現・登録上限・レコード設計・Lexicon設計 | [Lexicon・データ設計](02-lexicon.md) |
| キャッシュ・インデックス・表示補助データの保存方式、DB設計 | [全体構成](01-architecture.md)・[データフロー設計](05-data-flow.md) |

## このバンドルについて

- [OKF (Open Knowledge Format) v0.1](https://github.com/GoogleCloudPlatform/knowledge-catalog/blob/main/okf/SPEC.md) に準拠する。`index.md` と `log.md` は予約ファイルであり、それ以外の各`.md`はfrontmatter（`type` 必須）を持つ。
- 設計の変更時は該当ファイルを更新し、[log.md](log.md) に日付つきで記録する。
- 要件と矛盾する設計をしない。要件側の変更が必要な場合は、先に[要件定義書](../docs/index.md)を更新する。
