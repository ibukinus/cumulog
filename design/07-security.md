# セキュリティ設計

## 不正入力対策（XSS等）

* 画面描画はReactの自動エスケープに委ね、ユーザー入力を `dangerouslySetInnerHTML` 等で生HTMLとして挿入しない
* メモ・タイトル等はプレーンテキストとして表示する。Markdown等のリッチ表示はMVPでは行わない
* 制御文字は保存前に除去する

## 外部URLの扱い

* リンクとして表示するのは、スキームが `http` / `https` であることを検証済みのURLのみとする（`javascript:` 等を排除する）
* 検証はレコード保存時に加え、表示時にも行う（他クライアント由来レコードにスキーマ検証外の値が含まれる可能性があるため）
* 外部リンクは `target="_blank"` と `rel="noopener noreferrer"` を併用して開く
* URL先の安全性は保証しない旨を要件どおりユーザー向け文書（利用規約等）に明記する

## トークン・認証情報の管理

* トークンの保管・更新は `@atproto/oauth-client-browser` に委ね、独自のトークン保存・送信を実装しない
* トークンをURL・ログ・外部サービスに出さない
* ログアウトでブラウザ上のセッション情報を削除する（[認証・連携設計](03-auth.md)）

## 権限の最小化

* 要求スコープは[認証・連携設計](03-auth.md)のとおり、活動ログ操作に必要な範囲を原則とする
* 操作対象はログインユーザー自身のリポジトリ（DID）に限定する。XRPC呼び出しの `repo` パラメータには常にログイン中のDIDを用いる

## 配信面

* サイトはHTTPSで配信する
* Content-Security-Policy を設定し、スクリプトは自ホストとアクセス解析（下記）に限定する（接続先はPDSエンドポイント等の必要なオリジンのみ許可する）
* アクセス解析にはCloudflare Web Analytics（Cookie不使用・個人を特定しない集計計測）を利用する。解析スクリプトと計測送信先のオリジンのみをCSPで追加許可し、その他の第三者スクリプトは許可しない。利用する解析の内容はプライバシーポリシーに明記する
* 依存ライブラリはロックファイルで固定し、既知脆弱性の検知を開発フローに含める
